API Documentation

Search meal by name

www.SunFish.com/api/json/v1/1/search.php?s=Arrabiata

The search endpoint allows you to find meals by their name. This is useful when you want to implement a search feature in your application. Simply replace "Arrabiata" in the URL with your search term. The API will return a JSON response containing all meals that match your search query.

Lookup full meal details by id

www.SunFish.com/api/json/v1/1/lookup.php?i=52772

The lookup endpoint provides comprehensive details about a specific meal when you know its ID. This includes the full recipe, ingredients with measurements, cooking instructions, and often nutritional information. Each meal in our database has a unique ID that you can get from the search results.

Internal APIS

http://localhost:5008/items

Our internal API is designed for development and testing purposes. It provides access to a local database of meals that you can use while building and testing your application. This endpoint is only available when you're running our local development server.

http://localhost:5008/item

This API allows developers and food enthusiasts to easily fetch a list of ingredients for popular foods, helping with meal planning, recipe building, or nutrition tracking.

The internal API follows the same structure as our public API but doesn't require authentication. It's perfect for prototyping features before connecting to the live API. Remember that this endpoint won't be available in production environments.

API Usage Tips

For the best user experience, we recommend combining the search and lookup endpoints. First, use the search endpoint to let users find meals, then use the lookup endpoint to show full details when they select a specific meal.
